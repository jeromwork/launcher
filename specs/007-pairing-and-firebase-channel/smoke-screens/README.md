# Spec 007 — Managed-side smoke screens

Captured 2026-05-11 on `Medium_Phone_API_36.1` (Pixel-class, 1080×2400,
API 36.1) via the `android-emulator` skill, mockBackend flavor APK
(`com.launcher.app.mock`).

## What this exercises

This is the **partial** T110 smoke — just the Managed-side flow, since
the realBackend side requires a Firebase project + deployed Worker that
are not provisioned on this machine. The full two-emulator T110 (admin
scan + consent + push delivery + SC-003 timing) stays deferred and is
documented in `tasks.md §T110` plus `perf-checkpoint.md` §T106/SC-003.

## Screens

### Idle — `pairing-idle.png`

`PairingState.Idle`. "Remote control" heading, body explaining the
feature in plain language ("Let a trusted family member view and adjust
this device from their phone."), single "Turn on" tap target. Both
text-size and tap-target hit Article VIII §7 senior-safe budgets at a
glance: heading is bold ~28sp, body ~18sp, button height 56dp.

### Waiting-for-claim — `pairing-qr.png`

`PairingState.WaitingForClaim`. Captured immediately after tapping
"Turn on" — proves the FSM transition fires and `QrBitmapGenerator` +
`QrDisplayScreen` render correctly.

What renders:
- Heading "Show this code to your helper" (~24sp semibold).
- Body explaining what to do.
- Black-on-white QR encoding the deep link `launcher://pair?token=HEFPWM&v=1`
  (token freshly generated by `PairingService.generateToken` — note the
  `[A-HJ-NP-Z2-9]{6}` alphabet: H, E, F, P, W, M are all in range).
- Token text "Token: HEFPWM" — fallback for users who can't or won't
  scan, can read 6 chars out loud.
- Countdown "Time left: 4 min 57 sec" + LinearProgressIndicator that
  drains over the 5-min TTL.
- "Cancel" senior-safe button (56dp, fillMaxWidth, 18sp label).

No crashes in logcat; PairingActivity displayed in ~9.5 s cold start
(spec budget 10 s, SC-001).

## What was NOT verified

- **Admin side**: T089 scanner not implemented (deferred to spec 008).
- **Claim transaction over real Firestore**: T097/T098 (deferred — JDK 21
  + emulator). Domain logic covered by `PairingServiceTest` and
  `PairingEndToEndTest` against `FakeRemoteSyncBackend`.
- **FCM push delivery + SC-003 timing**: T110 full path — needs both
  emulators + realBackend Firebase config + deployed Worker.
- **Unbind UI flow**: not exercised on emulator in this session; covered
  by `PairedStatusSection` Compose code and `decline_clears_all_data`
  domain test.
- **Russian locale**: ru-RU strings exist in
  `app/src/main/res/values-ru/strings_pairing.xml`; emulator was running
  default en-US — screenshots therefore show English copy. RU rendering
  not explicitly tested on emulator but Compose `stringResource` picks
  the right resource bundle automatically.

## Reproduce

```sh
# Start the emulator (any single AVD; this run used Medium_Phone_API_36.1).
"$LOCALAPPDATA/Android/Sdk/emulator/emulator.exe" \
  -avd Medium_Phone_API_36.1 -port 5554 -no-boot-anim

# Build + install + launch.
./gradlew :app:installMockBackendDebug
"$LOCALAPPDATA/Android/Sdk/platform-tools/adb.exe" -s emulator-5554 \
  shell am start -n com.launcher.app.mock/com.launcher.app.ui.pairing.PairingActivity

# Screenshot.
"$LOCALAPPDATA/Android/Sdk/platform-tools/adb.exe" -s emulator-5554 \
  exec-out screencap -p > pairing-idle.png
```
