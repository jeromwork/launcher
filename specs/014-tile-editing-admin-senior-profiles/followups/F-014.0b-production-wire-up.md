## Architectural decisions to make **до** F-014.0b implementation

### Decision 1: Cross-module composable scope

F-014 composables в `:app/edit/` используют `androidx.compose.ui.res.stringResource(R.string.f014_*)`. Production wire-up в `:core/commonMain` требует Compose Multiplatform Resources (`launcher.core.generated.resources.Res`).

**Вариант A**: Migrate F-014 strings в `core/commonMain/composeResources/values/strings_spec014.xml` + переписать composables в `:core/commonMain/.../ui/edit/` через CMP resources API.

- ✅ Чисто, consistent с existing pattern (spec 008 sync UI, spec 009 EditorScreen).
- ❌ Migration effort: 7 composables + 30 string keys + ~6 plurals.
- ❌ Loses some Material 3 icons (`Icons.Filled.Add` доступен только в `:app` без material-icons-extended dep в `:core`).

**Вариант B**: Оставить F-014 в `:app/edit/`, обернуть HomeScreen/FlowScreen через `topSlot` / `bottomSlot` injection pattern (как `topSlot = HomeBannerHost` для спеки 006).

- ✅ Не требует migration.
- ❌ FlowScreen tiles нужно знать про edit mode (long-press, jiggle) — это **внутренний** аспект, не injectable через slot.
- ❌ Создаёт impedance — каждое расширение FlowScreen требует slot injection from outside.

**Вариант C**: Hybrid — domain composables (TileContextMenu, ConflictSnackbar, ConflictAcknowledgmentRequest) живут в commonMain; UI host (EditModeHost wrapper) — в `:app`.

- ✅ Domain-driven layering.
- ❌ Сложность для junior contributors.

**Рекомендация для F-014.0b**: **Вариант A** (migrate в commonMain). Это consistent с проектным конвенцией, и F-014.1 (server backup) + F-014.2 (encryption) тоже будут нуждаться в commonMain scope.

### Decision 2: EditorComponent (spec 009) vs EditModeComponent (F-014)

Сейчас два разных Decompose component для editing:
- `EditorComponent` (спека 009) — admin remote editing с merge UI, history snapshot, publish pipeline.
- `EditModeComponent` (F-014) — profile-aware edit mode с conflict asymmetry per Q7.

**Вариант A**: Merge — расширить `EditorComponent` функциональностью F-014 (profile selection, conflict asymmetry). EditModeComponent удалить.

- ✅ Один источник истины.
- ❌ Trogает spec 009 — все спеки 009 tests могут регрессировать.

**Вариант B**: Compose — `EditorComponent` wraps `EditModeComponent` через delegation.

- ✅ EditModeComponent остаётся isolated.
- ❌ Двойная state management — bugs.

**Вариант C**: Keep separate — `EditModeComponent` для US1 (admin self-edit Workspace) и US3 (senior local edit Simple Launcher); `EditorComponent` для US2 (admin remote edit). Каждый knows про свой profile.

- ✅ Чёткое разделение по US.
- ❌ Code duplication для conflict handling.

**Рекомендация для F-014.0b**: **Вариант C** или **B**. Финальный выбор — за clarify pass перед F-014.0b implementation.
