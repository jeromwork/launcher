# UX Checklist: WhatsApp Contact Tiles via Communication Shell

**Purpose**: Проверка качества UX-требований в спецификации (полнота, ясность, согласованность, измеримость), а не проверка реализации.  
**Created**: 2026-04-02  
**Feature**: [spec.md](./spec.md)

## Полнота требований

- [x] CHK001 Определены ли UX-требования для всех ключевых экранов потока (плитка, экран подтверждения, возврат в Launcher)? [Completeness, Spec §In Scope, §User Scenarios]
- [x] CHK002 Зафиксированы ли требования к визуальной структуре плитки (верх: фото+имя, низ: две кнопки) без пропусков по элементам? [Completeness, Spec §Clarifications, §FR-002, §FR-003]
- [x] CHK003 Указано ли, какие UX-состояния обязательны при недоступном переходе в WhatsApp? [Completeness, Spec §FR-017, §FR-018, §FR-019]
- [x] CHK004 Определены ли требования к UX при отмене действия до handoff, включая ожидаемое состояние домашнего экрана? [Completeness, Spec §FR-016, §User Story 3]

## Ясность требований

- [x] CHK005 Однозначно ли различаются термины "contact tile", "action confirmation screen" и "return context" во всех разделах? [Clarity, Spec §Key Entities]
- [x] CHK006 Ясно ли сформулировано, что действия в плитке строго ограничены кнопками `Call` и `Video` (без скрытых альтернатив)? [Clarity, Spec §FR-003, §FR-004]
- [x] CHK007 Уточнено ли в требованиях, какой именно "expected state" должен быть восстановлен после возврата? [Ambiguity, Spec §FR-015, §Stable Home State]
- [x] CHK008 Исключены ли расплывчатые UX-формулировки без критериев (например, "predictable") или они операционализированы? [Ambiguity, Spec §SC-003, §SC-004, §SC-005]

## Согласованность требований

- [x] CHK009 Согласованы ли между собой In Scope и Functional Requirements по структуре увеличенной плитки и действиям `Call`/`Video`? [Consistency, Spec §In Scope, §FR-002, §FR-003]
- [x] CHK010 Нет ли конфликта между требованием явного подтверждения и сценариями запуска действий с плитки? [Consistency, Spec §FR-005, §FR-007, §User Story 1]
- [x] CHK011 Согласованы ли edge-case требования по множественным нажатиям с защитой от дублей запуска? [Consistency, Spec §Edge Cases, §FR-020, §FR-021]

## Качество критериев приемки

- [x] CHK012 Можно ли объективно оценить UX-успех для каждого ключевого шага пути пользователя, а не только для итогового перехода? [Acceptance Criteria, Spec §User Scenarios, §SC-001, §SC-004, §SC-005]
- [x] CHK013 Привязаны ли метрики времени и успешности к конкретным UX-моментам (подтверждение, переход, возврат) без двусмысленности? [Measurability, Spec §SC-001, §SC-002, §SC-003]
- [x] CHK014 Указаны ли качественные UX-критерии для различения успешного и проблемного восстановления состояния после возврата? [Gap, Spec §FR-015, §FR-039, §SC-003]
