# Performance Accessibility Testing Checklist: WhatsApp Contact Tiles via Communication Shell

**Purpose**: Проверка качества требований по производительности, доступности и тестируемости UX/flow-потока (Launcher → WhatsApp → Launcher).  
**Created**: 2026-04-02  
**Feature**: [spec.md](./spec.md)

## Performance Requirements

- [x] CHK001 Указаны ли ограничения на время между подтверждением действия и запуском WhatsApp, чтобы обеспечить предсказуемую производительность? [Clarity, Spec §SC-002]
- [x] CHK002 Определено ли допустимое время восстановления Launcher после возврата, чтобы соблюсти ожидания пользователя? [Measurability, Spec §SC-003]
- [x] CHK003 Описаны ли требования к устойчивости потока при медленных handoff-ах (например, отображение прогресса / таймауты)? [Gap, Spec §FR-017, §FR-033, §User Story 4]

## Accessibility Requirements

- [x] CHK004 Определены ли требования к размеру и контрасту двух крупных кнопок (`Call`, `Video`) на плитке? [Clarity, Spec §FR-003, §FR-030]
- [x] CHK005 Задано ли требование по читабельности фото+имени (шрифт, контраст) на плитке? [Gap, Spec §FR-002, §FR-030]
- [x] CHK006 Указано ли, как подтверждение и возврат будут доступны с клавиатуры или дескрипторами? [Coverage, Spec §FR-032]
- [x] CHK007 Определены ли требования к аудиозаписям/зрительному подтверждению для пользователей с ограниченным зрением? [Gap, Spec §FR-030, §FR-032, §FR-033]

## Testing Requirements

- [x] CHK008 Можно ли по требованиям воспроизвести сценарий отказа запуска WhatsApp и описать ожидаемое состояние Launcher? [Coverage, Spec §User Story 4, §FR-017, §FR-018, §FR-019]
- [x] CHK009 Ясно ли, какие проверяемые результаты нужны для возврата из WhatsApp (сохранённый контекст, состояние UI)? [Clarity, Spec §User Story 2, §FR-014, §FR-015]
- [x] CHK010 Указано ли, как тестировать защиту от двойных нажатий (например, блокировка UI)? [Coverage, Spec §FR-020, §FR-021]
- [x] CHK011 Определены ли метрики успешности подтверждения (например, правильная пара контакт+действие) для acceptance test? [Measurability, Spec §SC-001]
- [x] CHK012 Уточнено ли, как будет проверяться реставрация экрана при разных методах возврата (Back/Home/Switcher)? [Coverage, Spec §FR-022]
