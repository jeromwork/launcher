# Performance Accessibility Testing Checklist: WhatsApp Contact Tiles via Communication Shell

**Purpose**: Проверка качества требований по производительности, доступности и тестируемости UX/flow-потока (Launcher → WhatsApp → Launcher).
**Created**: 2026-04-02
**Feature**: [spec.md](../spec.md)

**Note**: Чеклист фокусируется на том, что требования сами по себе достаточно ясны, измеримы и покрывают важные нефункциональные сценарии, без проверки реализации.

## Performance Requirements

- [ ] CHK001 Указаны ли ограничения на время между подтверждением действия и запуском WhatsApp, чтобы обеспечить предсказуемую производительность? [Clarity, Spec §SC-002]
- [ ] CHK002 Определено ли допустимое время восстановления Launcher после возврата, чтобы соблюсти ожидания пользователя? [Measurability, Spec §SC-003]
- [ ] CHK003 Описаны ли требования к устойчивости потока при медленных handoff-ах (например, отображение прогресса / таймауты)? [Gap, Spec §User Story 1, Spec §FR-010]

## Accessibility Requirements

- [ ] CHK004 Определены ли требования к размеру и контрасту двух крупных кнопок (`Call`, `Video`) на плитке? [Clarity, Spec §Clarifications, Spec §FR-003]
- [ ] CHK005 Задано ли требование по читабельности фото+имени (шрифт, контраст) на плитке? [Gap, Spec §FR-002]
- [ ] CHK006 Указано ли, как подтверждение и возврат будут доступны с клавиатуры или дескрипторами? [Coverage, Spec §FR-004, Spec §FR-008]
- [ ] CHK007 Определены ли требования к аудиозаписям/зрительному подтверждению для пользователей с ограниченным зрением? [Gap, Non-Functional]

## Testing Requirements

- [ ] CHK008 Можно ли по требованиям воспроизвести сценарий отказа запуска WhatsApp и описать ожидаемое состояние Launcher? [Coverage, Spec §FR-010, Spec §Edge Cases]
- [ ] CHK009 Ясно ли, какие проверяемые результаты нужны для возврата из WhatsApp (сохранённый контекст, состояние UI)? [Clarity, Spec §User Story 2, Spec §FR-008]
- [ ] CHK010 Указано ли, как тестировать защиту от двойных нажатий (например, блокировка UI)? [Coverage, Spec §FR-011]
- [ ] CHK011 Определены ли метрики успешности подтверждения (например, правильная пара контакт+действие) для acceptance test? [Measurability, Spec §SC-001]
- [ ] CHK012 Уточнено ли, как будет проверяться реставрация экрана при разных методах возврата (Back/Home/Switcher)? [Coverage, Spec §FR-012]

## Combined Non-Functional Coverage

- [ ] CHK013 Указано ли, что ручки производительности и доступности не требуют фоновых наблюдателей? [Restrictions, Spec §Constraints]
- [ ] CHK014 Проверено ли, что mock-конфигурация поддерживает тестируемые UX-сценарии без дополнительных данных? [Assumption, Spec §FR-013]
- [ ] CHK015 Описаны ли критерии успеха, повторяемые в тестах (связь успеха UX и указанных meta-ограничений)? [Traceability, Spec §Success Criteria, Spec §Clarifications]
