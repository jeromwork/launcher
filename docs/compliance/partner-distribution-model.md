# Partner Distribution Model

## Цель
Фиксировать отдельные правила дистрибуции через партнеров.

## Рассматривать для каждого партнерского сценария
- тип партнера,
- регион,
- канал установки,
- ownership billing,
- ownership support,
- кастомизация сборки,
- branding/co-branding,
- legal/SLA obligations,
- update path,
- observability and support obligations.

## Важный принцип
Партнерская дистрибуция — не просто “еще один магазин”.
Она может менять:
- сборку,
- конфигурацию,
- entitlement/package model,
- legal wording,
- support processes.

## Следствие
Если фича или сборка затрагивает partner channel, это должно быть отражено:
- в `plan.md`,
- в этом документе,
- при необходимости в отдельном ADR.
