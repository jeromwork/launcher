## Что это простыми словами

Главный экран — сетка, например 2×3. Часть ячеек занята (WhatsApp, Telegram, контакт), часть пуста. Вопросы:
1. Как показывать пустую ячейку? (плюсик → список «что добавить»)
2. Что происходит при нажатии «добавить» и откуда берётся список вариантов?
3. Является ли «пустая ячейка» частью пресета при шаринге, или пустота — просто отсутствие компонента?

## Как это решено в индустрии (research 2026-07-16)

- **Android Launcher3 / Pixel Launcher / Nova**: элементы хранятся плоскими строками БД (favorites) с координатами (container, screen, cellX, cellY). Пустая ячейка = отсутствие строки, отдельной сущности «пустой слот» нет. «Добавить» — жест/кнопка UI, не элемент данных.
- **iOS SpringBoard**: аналогично — список элементов с позициями; пустоты вычисляются.
- **Шаринг лейаутов** (Nova backup): передаются элементы с позициями; пустые ячейки приезжают «бесплатно» как отсутствие.

**Рекомендация**: пустой слот НЕ делать компонентом. Пустота = отсутствие; плюсик = состояние UI. Если пресету понадобится «зарезервированная пустая ячейка с подсказкой» — это аддитивное поле позиции на AppTile (`cellPosition`) + подсказки в hintFlow, не новый вид сущности.

## Что входит технически (для AI-агента)

- UI пустых ячеек: плюсик в свободных ячейках сетки `layoutKey` (senior-safe: крупный, не пугающий).
- Список «что добавить»: из пула (`availableTemplates` семантика — приложения, контакты, будущие календарь/виджеты), фильтр по установленности на устройстве.
- `addFlow` на profile-пути: сейчас заглушка `TODO(profile-add-flow)` (TASK-127 parity) — реализовать добавление ProfileComponent через ProfileStore + ReconcileEngine (RunMode.Single).
- Опционально (решить в clarify): явные позиции `cellPosition` на AppTile — аддитивное поле, нужно только если пользователь может расставлять плитки в произвольные ячейки.
