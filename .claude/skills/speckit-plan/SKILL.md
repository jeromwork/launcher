# Orchestrator: speckit-plan

Bridges **clarified spec → planned implementation**. Translates spec.md into architectural decisions per [`/.specify/templates/plan-template.md`](/.specify/templates/plan-template.md), and validates against the constitution.

---

## When to invoke

- `spec.md` exists, has passed `speckit-clarify` (no major open checklist items).
- No `plan.md` yet OR plan.md is being significantly revised.

## When NOT to invoke

- `spec.md` has open `[NEEDS CLARIFICATION]` markers — go back to `speckit-clarify`.
- Trivial change (a new FR consistent with existing plan) — edit plan.md directly.

---

## Procedure

### Step 1 — Inventory

Read spec.md. List:
- Every new module / port / data type to introduce.
- Every external dependency (vendor SDK, system API, persistence).
- Every wire format that will exist.
- Every UI surface.

### Step 2 — Generate plan.md

Use [`/.specify/templates/plan-template.md`](/.specify/templates/plan-template.md) as base. **Mandatory sections** (don't skip):

1. **Overview** — 2 sentences linking to spec.md.
2. **Architecture** — module map, port-adapter shape, data flow diagram (text or mermaid).
3. **Data model** — link to `data-model.md` (created in Step 4).
4. **Wire formats** — list each, link to `contracts/<name>.md`.
5. **Dependency impact** — new gradle deps, justified per Article XIII.
6. **Test strategy** — per CLAUDE.md rule §6 + §7: contract, integration, fake-adapter, fitness functions.
7. **Risks** — OEM, lifecycle, permissions, performance — explicit list with mitigation.
8. **Required Context Review** — every relevant file from `docs/governance`, `docs/adr`, `docs/product`, `docs/compliance` linked. Per Article XII §7.
9. **Constitution Check** — generated by `procedure-constitution-check`. **Required.**
10. **Rollout / verification** — how we know it works (perf checkpoint, smoke, fitness functions).

### Step 3 — Generate sub-artifacts

Conditionally:
- **`research.md`** — if any decision needed comparing alternatives (≥1 one-way door per CLAUDE.md §3). Skip for trivial plans.
- **`data-model.md`** — if any new persistent or wire-format data types. Skip otherwise.
- **`quickstart.md`** — if a new dev workflow, build setup, or test command is introduced.
- **`contracts/`** — one `.md` file per wire format / external interface. Use [`specs/002-whatsapp-tile-return/contracts/`](specs/002-whatsapp-tile-return/contracts/) as the format reference.

### Step 4 — Run Constitution Check

Invoke `procedure-constitution-check` with the just-written plan.md path. Inline its full report into plan.md as section "Constitution Check".

If any gate FAILs:
1. Stop generating other artifacts.
2. Report failures to user.
3. Either fix the plan, or ask user to amend the constitution / approve exception per Article XVII.

### Step 5 — Run targeted checklists

If `speckit-clarify` was already run, its checklists cover spec-level concerns. Now run **plan-level** subset:
- `checklist-domain-isolation` (re-run on plan, verify port/adapter shape).
- `checklist-wire-format` (re-run, verify schemaVersion in contracts).
- `checklist-meta-minimization` (re-run on plan abstractions specifically).

### Step 5b — Add novice summaries

**Mandatory.** For every artifact this orchestrator produced (`plan.md`, `research.md`, `data-model.md`, `quickstart.md`, every file in `contracts/`), invoke `procedure-add-novice-summary` to append a plain-Russian "for newcomers" section at the bottom.

### Step 6 — Report

```
SPECKIT-PLAN for specs/<id>/:
  Generated: plan.md (358 lines), research.md, data-model.md, contracts/{action,provider-registry}.md
  Skipped: quickstart.md (no new dev workflow)
  Constitution Check: 8/8 PASS (after 2 iterations)
  Plan-level checklists: domain-isolation 16/16 ✓, wire-format 17/18 ✓ (CHK-016 missing), meta-minimization 13/13 ✓
  Open issues: 1 — see wire-format checklist
  Next step: address CHK-016, then run speckit-tasks
```

---

## Sizing

- **Tiny spec**: plan.md only, no sub-artifacts, Constitution Check inline.
- **Standard spec**: plan.md + data-model.md + contracts/ + research.md if one-way doors.
- **Large spec**: full set + quickstart.md.

Default to **less artifact** unless evidence demands more. Article XII §3: "tasks.md must contain execution items, not architecture essays" — same applies to plan.md.
